Spark故障排除——shuffle file cannot find

Veröffentlicht am 2017-03-12 | in Spark |

shuffle file cannot find

错误描述：shuffle file cannot find

【原因】

1.map task所运行的executor挂掉了，executor里面的blockManager就挂掉了。

executor什么情况会挂掉?

1.堆内内存不足 OOM,设置参数–executor-memory 6G

2.堆外内存不足 OOM。默认每一个Executor 会申请堆外内存大小是Executor-memory的10%。

设置参数，如果是spark on yarn --conf spark.yarn.executor.memoryOverhead=2048 单位是M。如果是spark standalone --conf spark.executor.memoryOverhead=2048

Spark底层shuffle的传输方式是使用netty传输，netty在进行网络传输的过程会申请堆外
内存（netty是零拷贝），所以使用了堆外内存。

2.executor没有挂掉

blockManager之间的建立连接失败(比如map task 所运行的executor正在GC) 设置参数--conf spark.core.connection.ack.wait.timeout=60。

建立连接成功，在拉取数据的时候出现了问题(比如map task 所运行的executor正在GC)

3.reduce task在向Driver里面的MapOutputTracker获取shuffle file位置的时候出现了问题。
